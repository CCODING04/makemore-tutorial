## 2. 卡点清单

### 🔴 严重卡点（会停下来想很久 / 可能放弃）

#### 卡点 1：Part1 02 — Broadcasting 规则讲得太简略

```python
P /= P.sum(1, keepdims=True)
```

教程用了一个代码块解释 broadcasting，但**对于 PyTorch 新手来说，broadcasting 是个全新的概念**。教程只说了 `keepdims=True` 很重要，但没有解释：

- 为什么 `(27, 27) / (27, 1)` 会自动广播？
- 广播的规则到底是什么？（从右对齐、维度一致或一方为 1）
- 如果 `keepdims=False` 会发生什么？具体错误是什么？

**我的感受**：我其实不太确定这一步到底发生了什么，只能"照着敲"。后面 Part 2 大量使用 broadcasting（`C[X]` 的形状变化、`emb.view` 等），如果这里没弄明白，会越学越糊涂。

#### 卡点 2：Part2 02 — `emb.view(emb.shape[0], -1)` 的 `-1` 是什么？

```python
emb = C[X]           # (N, 3, 2)
emb_cat = emb.view(emb.shape[0], -1)  # (N, 6)
```

教程说了一句"`-1` 让 PyTorch 自动算第二维"，但：
- 为什么是 `(N, 6)` 不是别的？自动计算的规则是什么？
- `view` 和直接 reshape 有什么关系？
- 如果我不小心写了 `.view(-1, 6)` 会怎样？

虽然课后练习 Q2 提到了 `view` vs `reshape`，但这是**正向传播的核心步骤**，在教程正文中应该更详细。

#### 卡点 3：Part2 02 — `C[X]` 的高级索引操作没解释

```python
emb = C[X]  # (N, 3, 2)
```

这一行代码做了很多事：`X` 是 `(N, 3)` 的整数矩阵，`C` 是 `(27, 2)` 的矩阵，结果变成 `(N, 3, 2)`。这对 PyTorch 新手来说是**魔法**。

教程只说了"查表得到 3 个 2 维向量"，但缺少：
- 为什么整数索引可以一次查多个？
- `C[X]` 的形状推导规则是什么？
- 和 `F.one_hot(X) @ C` 的等价关系（教程 Q2 提到了，但应该放在正文里）

### 🟡 中等卡点（会停下来查资料 / 需要反复看）

#### 卡点 4：Part1 03 — 学习率 50 为什么能用这么大？

```python
W.data += -50 * W.grad  # 学习率 50
```

教程说"这个任务比较简单，可以用大学习率"，但没有解释：
- 为什么简单就可以用大学习率？
- 大学习率在什么情况下会出问题？
- 到了 Part 2 学习率变成 0.1，为什么差这么多？

这个概念跳跃对新手来说不太友好。

#### 卡点 5：Part2 01 — Train/Dev/Test 那段文字有口误

```
我们有三份名字数据：32033 + 3 + 1 = 32033+3+1。不对，让我重新说——
```

这是**写作时的自我纠错残留**，像草稿没删干净。虽然后面重新解释了，但作为一个"学生"，看到这段会困惑一下——作者在说什么？

#### 卡点 6：Part2 03 — 学习率搜索那段代码缺少上下文

教程展示了"学习率搜索"的代码和一张 loss vs 学习率的图，但：
- 没有解释 `lre = torch.linspace(-3, 0, 1000)` 为什么是在指数空间搜索
- 没有说最终推荐的学习率是多少
- 代码只是片段，没有完整脚本对应（`05_minibatch_training.py` 用的是固定 lr 调度，不是搜索）

#### 卡点 7：Part1 → Part2 过渡 — 字符映射不一致

Part 1 的字符映射：
```python
stoi = {s: i + 1 for i, s in enumerate(chars)}  # a=1, b=2, ...
stoi['.'] = 0
```

Part 2 的字符映射（在脚本中）：
```python
chars = ['.'] + chars  # '.' 放最前面
stoi = {s: i for i, s in enumerate(chars)}  # .=0, a=1, ...
```

结果一样，但写法不同。对于一个正在学习的学生，可能会花时间对比两种写法是否等价。建议统一。

### 🟢 轻微卡点（稍微想一想就过）

#### 卡点 8：Part2 03 — Embedding 可视化的"元音聚在一起"

教程说训练后"元音字母聚在一起"，但这是对**训练完的**模型来说的。学生在读到这里时可能还没训练模型，不知道自己在看什么。可以加一句"训练 20000 步后，你会看到..."

#### 卡点 9：Part1 02 — NLL 推导链略显密集

```
似然 = P(e|.) × P(m|e) × ...
log 似然 = log P(e|.) + log P(m|e) + ...
NLL = -log 似然
平均 NLL = NLL / bigram 总数
```

四步推导连在一起，虽然标记了🔑，但对概率论不熟的同学可能需要更多直觉解释。比如可以加一句："为什么要取负？因为我们习惯说「降低 loss」而不是「提高似然」。"

#### 卡点 10：作业 2 的 `evaluate` 函数签名在测试和作业描述中不一致

作业描述里的 `evaluate` 函数签名：
```python
def evaluate(X, Y, C, W1, b1, W2, b2):
```

但作业描述的"提示"代码中写的是：
```python
logits = mlp_forward(X, Y, C, W1, b1, W2, b2)  # 多了 Y 参数！
```

`mlp_forward` 不需要 `Y` 作为参数（它只做前向传播），所以这里的提示代码有 bug。
